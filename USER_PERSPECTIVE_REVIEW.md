# 用户视角 Code Review（Chase Application Status Viewer）

> Review date: 2026-04-14  
> Scope: popup 主流程（扫描、手动粘贴、结果展示、错误反馈）

## 总体结论

这个扩展的核心价值是明确的：让用户更快看到申请状态和后续动作。主路径（打开弹窗 → 一键扫描 → 看结果）短，信息层级基本清晰。  
但从“普通用户”角度，仍有一些关键体验风险：

1. **错误提示偏技术化且归因不清晰**（用户不知道下一步要做什么）。
2. **状态解释可能产生“过度确信”**（尤其是 “No Action Required” 文案）。
3. **可访问性不足**（hover-only 提示、颜色依赖、键盘可用性不足）。
4. **扫描失败场景缺少明确分流**（权限问题 / 页面上下文问题 / API 不可访问问题混在一起）。

## 用户旅程评估

### 1) 首次使用（新用户）

**优点**
- 顶部品牌与版本可见，建立基本信任感。
- “One-Click Scan”按钮突出，降低学习成本。
- 扫描失败后给了手动 JSON 指南，具备 fallback。

**问题**
- 指南内容仅在用户主动展开/失败后触发，首次用户可能仍不懂“为什么扫不到”。
- “Use on 'Application Status' page.” 过于简略，不含可点击示例路径或截图提示。

**建议**
- 在扫描按钮下增加 1 行“最小前置条件”提示（例如：需停留在具体状态页并刷新过一次）。
- 给“How to get JSON manually?”加入一键展开的默认策略（首次访问默认展开，后续记忆收起状态）。

### 2) 扫描失败场景

**优点**
- 有统一错误区域，不会静默失败。

**问题**
- 当前错误文案将多类失败统一为“Refresh & Try again”，可执行性弱。
- 用户无法区分：
  - 站点不对；
  - 当前 tab 无法注入脚本；
  - API 未命中；
  - JSON 结构不符合预期。

**建议**
- 按错误类型输出“下一步动作按钮”：
  - `Wrong Site` → “Open Chase status page”说明；
  - `NOT_FOUND` → “Show manual guide”；
  - 权限/脚本错误 → “Reopen extension + refresh page + check host permissions”。
- 在错误信息中避免过多内部术语（如 Script Error），改成用户动作导向文案。

### 3) 结果页（Dashboard）

**优点**
- 状态 Banner、关键参考号、更新时间、所需材料分区清楚。
- 支持多申请过滤，适合重度用户。

**问题**
- “No Action Required”容易被理解为“完全没事”，但后端字段缺失并不等于真实世界无动作。
- “Code: xxx”这类技术字段较多，普通用户不一定理解价值。
- 错误码区直接外链搜索，可能带来信息可信度噪声。

**建议**
- 将“No Action Required”改为更保守措辞（如“未检测到待补材料（以官方通知为准）”）。
- 对技术字段增加可折叠“高级信息”区，默认隐藏。
- 错误码搜索旁边加免责声明（搜索结果非官方结论）。

### 4) 可访问性与可用性

**问题**
- Tooltip 依赖 hover，不利于键盘和触屏用户。
- 一些状态主要靠颜色区分（红/绿/黄），对色弱用户不友好。
- 文本字号大量使用 10px/9px，信息密度高但阅读压力较大。

**建议**
- Tooltip 同时支持 focus（tab）触发，并加 aria 描述。
- 状态标签添加图标/文案双编码（不仅靠颜色）。
- 将关键正文最小字号提升到 11~12px，尤其错误和指导文案。

## 风险分级（按用户影响）

### P0（高优先，建议尽快）
1. 失败提示可执行性不足，导致用户卡死在“刷新试试”。
2. “No Action Required”措辞可能引发误解，存在信任风险。

### P1（中优先）
1. 可访问性（键盘/触屏/低视力）不足。
2. 技术字段默认暴露太多，干扰普通用户决策。

### P2（可迭代优化）
1. 首次使用引导可更“任务化”（一步一步）。
2. 错误码外链策略可增加官方资源优先级。

---

## 建议的下一个迭代（用户导向）

1. **重写错误体系（文案+分流）**：每类错误给明确下一步。  
2. **保守化关键结论文案**：减少“绝对判断”语气。  
3. **可访问性 quick win**：focus tooltip、提高最小字号、按钮 aria 标签。  
4. **信息分层**：默认展示“我该做什么”，高级字段收起。

---

## 一句话总结

从用户视角，这个产品已经“能用且有价值”，但仍需把“技术可行”提升为“决策可信、失败可恢复、更多人可用”。
